# Sesion Codex 2026-06-12

## Resumen

Se ajusto la documentacion para dejar `carabuela` como tenant principal de
prueba/validacion mientras el sistema se estabiliza. Los demas tenants se
agregaran despues, cuando el flujo este listo.

Tambien se corrigio un problema visual del layout: al entrar a paginas con
tablas anchas, como Abonados, el menu lateral podia achicarse. La solucion deja
el menu con ancho estable y permite que las tablas hagan scroll horizontal
dentro del contenido.

## Tenant de prueba

Decision actual:

- `carabuela` queda como tenant de prueba/validacion.
- Las pruebas tenant deben hacerse con URLs prefijadas, por ejemplo
  `/carabuela/abonados/`.
- `default` queda como base legacy/de pruebas generales.
- Los nuevos tenants se agregaran progresivamente cuando el sistema este
  estable.

## Layout y tablas

Problema detectado:

- En Medidores el menu lateral mantenia su ancho esperado.
- En Abonados, al existir una tabla con mas columnas, el area de contenido
  forzaba el layout y el menu lateral se achicaba.

Correccion aplicada:

- En `base.html`, el menu lateral usa `md:shrink-0`.
- En `base.html`, el contenedor principal usa `min-w-0`.
- En `app/static/css/app.css`, se agrego la clase `tabla-scroll`.
- Las tablas operativas existentes se envolvieron con
  `tabla-scroll overflow-x-auto`.
- Las tablas sueltas de detalle de factura y reporte mensual tambien quedaron
  dentro de un contenedor de scroll.

Regla para futuras tablas:

```html
<div class="tabla-scroll overflow-x-auto">
    <table class="w-full text-sm">
        ...
    </table>
</div>
```

Con esto, si una tabla crece en columnas, el scroll horizontal queda dentro de
la tabla y el menu lateral no cambia de ancho.
